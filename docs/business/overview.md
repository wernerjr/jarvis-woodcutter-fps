## 1. Objetivos de negócio
1. Validar um gameplay loop de sobrevivência/crafting acessível via navegador.
2. Entregar experiência multiplayer leve com baixa fricção de entrada (guest).
3. Criar base técnica evolutiva para novos modos, conteúdo e economia.

## 2. Público-alvo e plataformas
- **Público-alvo primário**: jogadores casuais e mid-core que gostam de crafting/progressão.
- **Plataforma de lançamento inicial**: web desktop (browser com WebGL).
- **Plataformas futuras (potencial)**: web mobile/tablet (com adaptações de UX/performance).

## 3. Modelo de monetização
Status atual: **sem monetização implementada no código**.

Opções futuras compatíveis com o design:
1. Cosméticos visuais (skins, temas de UI).
2. Battle pass sazonal com recompensas não pay-to-win.
3. Servidores/mundos premium com QoL adicional.

## 4. Métricas de sucesso
- Técnica
  - Latência WS estável e baixa taxa de erro em APIs críticas.
  - Tempo de carregamento inicial competitivo para web.
- Produto
  - Retenção D1/D7.
  - Tempo médio de sessão.
  - Conversão do tutorial natural (entrar -> coletar -> craftar).
- Negócio
  - Crescimento de MAU/WAU.
  - CAC orgânico (se houver aquisição paga futura).
  - Engajamento por atualização de conteúdo.
